# IliaGPT

## Overview
IliaGPT is an AI-powered chat application designed as an intelligent assistant for autonomous web browsing and document creation. Its core purpose is to offer a versatile platform for AI-driven tasks, including economic data analysis, multi-intent prompt processing, and professional document generation. The ambition is for IliaGPT to become a leading AI assistant for productivity.

## User Preferences
Preferred communication style: Simple, everyday language.

## System Architecture
IliaGPT is built with a monorepo structure, separating client, server, and shared components, ensuring type safety with Zod schemas.

### UI/UX
The frontend, developed with React, TypeScript, and Vite, utilizes shadcn/ui (Radix UI) and Tailwind CSS for a modern, themable interface supporting light/dark mode. Key features include chat folders, command history, draft auto-save, suggested replies, conversation export, message favorites, prompt templates, PWA support, and keyboard shortcuts. Content rendering supports Markdown, code highlighting, and mathematical expressions using KaTeX. Data visualization is provided by Recharts, ECharts, and TanStack Table, with a multi-layer graphics rendering system supporting SVG (D3.js), Canvas 2D, and 3D (Three.js). Performance is optimized through message virtualization, memoization, lazy loading, streaming UX, and robust error handling. Security and accessibility are addressed via DOMPurify sanitization, frontend rate limiting, MIME type validation, and ARIA support. A file preview system provides consistent styled cards for various file types (PDF, DOCX, XLSX, text) and dedicated preview modals, with PDF rendering via `react-pdf` and DOCX/XLSX/text previews served as HTML from the backend.

### Technical Implementation
The backend uses Node.js and Express.js, featuring an LLM Gateway for AI model interactions with multi-provider fallback, request deduplication, streaming recovery, token usage tracking, circuit breakers, and response caching. Agentic tool calling is supported via `streamChatWithTools()`, including OpenAI-compatible function calling with automatic tool execution loops. An ETL Agent automates economic data processing. The system incorporates a Multi-Intent Pipeline for complex user prompts and a PARE System (Prompt Analysis & Routing Engine) for production-grade document processing across various formats (PDF, DOCX, XLSX, PPTX, CSV, TXT), including per-document citations and a defense-in-depth architecture. A Document Generation System uses LLM orchestration to create Excel and Word files, including professional CV/Resume generation. The Spreadsheet Analyzer Module offers AI-powered analysis, an LLM agent for Python code generation (with AST-based security validation), and a secure Python sandbox.

Core agentic tools from OpenClaw v2026.3.1 are integrated directly, including `bash` (hardened with 30+ blocked command patterns, ulimit resource limits, real-time stdout streaming, and audit logging), `web_fetch` (structured content extraction with heading/list/table preservation, proper redirect chain following up to 5 hops, native JSON API support, and friendly error messages), `web_search` (DuckDuckGo), `read_file`, `write_file`, `edit_file`, `list_files` (with path traversal protection), `run_code` (isolated Python/Node execution with safety filters and timeout), `process_list` / `port_check` (process management with safety checks), `rag_index_document` (deep document chunking with hybrid BM25+vector search), and `openclaw_rag_search` (semantic knowledge base search with chatId exclusion filtering). These tools are dispatched via `executeOpenClawToolCall` and merged into the agent executor's tool array. The agent executor uses OpenAI-compatible API calls through OpenRouter, routing via `OPENAI_BASE_URL` with `minimax/minimax-m2.5` as the default model.

**AGENTOS-ASI Cerebro Pipeline** (`server/agent/cerebro/`): Neuro-symbolic hierarchical agent with Planner→Executor→Critic→Judge stages. The PlannerAgent decomposes complex requests into a DAG of subtasks with dependencies, priorities, and tool hints (auto-enriched by `capabilityDiscovery`). Subtasks execute in parallel waves respecting dependencies. The CriticAgent evaluates each subtask (completeness, grounding, coherence) and decides accept/retry/backtrack (max 2 retries). The JudgeAgent provides final evaluation with quality/grounding/coherence scores. A WorldModel (pure TypeScript state machine) tracks files modified, commands run, discoveries, and environment state. SSE events (`plan_update`, `subtask_start`, `subtask_complete`, `critic_result`, `judge_verdict`) stream pipeline progress to the frontend `AgentRunTimeline` component.

**Multi-Model Router** (`server/agent/modelRouter.ts`): Policy-based routing across 6 model tiers (minimax, openai/gpt-4o-mini, openai/gpt-4o, anthropic/claude-3.5-sonnet, deepseek, google/gemini-2.5-flash) via OpenRouter. Routes by task complexity (simple/moderate/complex) and type. Per-model circuit breaker (3 failures = open, 60s recovery). Env-configurable: `AGENT_MODEL`, `AGENT_FALLBACK_MODELS`, `AGENT_COST_CEILING_USD`.

**Token Budget Manager** (`server/agent/budgetManager.ts`): Tracks prompt/completion tokens, estimated cost per run. Configurable budgets: max tokens (default 100k), cost ceiling (default $1.00), iteration limit. Warns at 80%, halts at 100%. Emits `budget_update` SSE events with real-time usage stats.

**Prompt Injection Hardening** (`server/agent/security/inputSanitizer.ts`): Detects instruction overrides, role hijacking, data exfiltration, system prompt extraction, encoding attacks, delimiter injection. Severity scoring (low/medium/high). High-severity blocks request with user-friendly message. Content boundary markers delimit system/user/tool content. Output sanitization strips leaked prompts, API keys, internal markers.

**Capability Discovery** (`server/agent/capabilityDiscovery.ts`): Maps task descriptions to tool combinations across 8 categories (research, file-ops, web-automation, code-analysis, data-processing, document-creation, system-ops, memory-retrieval). Suggests tool chains, identifies missing tools, and provides alternatives. Wired into cerebro planner for automatic tool hint enrichment.

The agent loop features production-grade reliability: exponential backoff retry for transient LLM API failures (429/503/502), a CircuitBreaker that trips after 3 consecutive tool failures (including logical errors), parallel tool execution when the model returns multiple tool_calls simultaneously, high-risk action detection with SSE `tool_requires_confirmation` events and trace logging, and automatic memory persistence of agent runs via RAG indexing.

**Computer-Control-Plane** (`server/agent/computerControl/`): Full terminal/command governance system with: `riskClassifier.ts` (classifies shell commands into 4 levels: safe/moderate/dangerous/critical, detects sudo, rm -rf, fork bombs, system mods, network changes, package installs), `killSwitch.ts` (emergency stop with arm/disarm, aborts all running tool calls via AbortControllers, clears execution queue, prevents new runs), `sessionRecorder.ts` (full audit trail of all tool executions with timestamps, inputs, outputs, risk classifications, exit codes), `index.ts` (unified ComputerControlPlane wiring risk classifier + kill switch + session recorder with single `evaluateCommand()` entry point).

**Skills Kernel** (`server/agent/skillsKernel.ts`): Enhanced tool management layer with RBAC/ABAC (permission checks per tool by user role free/pro/admin with context-based conditions), Idempotency (SHA-256 hash of inputs, cache results per run, TTL-based expiry, max 1000 entries), Compensation/Rollback (register undo functions per tool call, execute in reverse on failure cascade), Skill Composition (combine tools into higher-level skills with input/output mappers, conditional execution, auto-rollback). Unified `executeWithKernel()` chains RBAC→idempotency→execution→cache→compensation.

**Enhanced Memory System** (`server/agent/memory/`): `episodicMemory.ts` (per-session interaction records with importance scoring based on type/keywords, exponential decay with access-count boost, configurable max entries), `projectMemory.ts` (cross-session project knowledge using `agentMemoryStore` table, 7 categories: architecture_decision/file_purpose/user_preference/project_convention/dependency_info/known_issue/workflow_pattern, upsert logic, TTL expiry), `memoryManager.ts` (unified retrieval across episodic+project, consolidation of duplicates, maintenance cycles with decay/cleanup, privacy controls per user with PII redaction, `forgetByUser()` for full data deletion).

**Event Sourcing** (`server/agent/eventSourcing.ts`): CQRS pattern with deterministic replay using `agentModeEvents` table. EventStore with `appendEvent()` (command-side), `getEventsForRun()` (query-side), `replay()` (deterministic state reconstruction), `getStateAtEvent()` (point-in-time). Event types: CommandReceived, PlanCreated, SubtaskStarted, ToolCalled, ToolCompleted, CriticEvaluated, JudgeVerdict, StateTransition, SnapshotTaken, ReplanTriggered. Automatic snapshots every 20 events. CommandBus + QueryBus for CQRS separation. API: `GET /api/agent/runs/:runId/replay` (full event stream with snapshots), `GET /api/agent/runs/:runId/state` (point-in-time state).

**OpenTelemetry Tracing** (`server/agent/tracing.ts`): Lightweight OTel-compatible tracing without external dependencies. Span hierarchy: Run→Phase→Step→ToolCall. Each span: traceId, spanId, parentSpanId, name, startTime, endTime, status, attributes, events. RunTracer class manages spans per run. Metrics: latency histograms (bucketed), token counts, error rates, tool call frequency, phase breakdowns. OTel JSON export via `toOTelExport()`. EventBus integration via `setupEventBusTracing()`. API: `GET /api/agent/runs/:runId/traces` (span tree + metrics, supports `?format=otel`).

**RAG++ Service** (`server/services/ragService.ts`): RAGFlow-style document understanding with automatic chunking (800-char chunks with 200-char overlap), key phrase extraction, bigram/trigram local embeddings, and hybrid scoring (50% vector + 35% BM25 + 15% key phrase boost). Enhanced with: query rewriting (compound decomposition + synonym expansion), LLM-based reranking, multi-hop retrieval (auto-refine if scores low, up to 3 hops), evidence packs with citations (source, chunk index, relevance score, provenance), and freshness/TTL decay (7-day half-life).

The frontend ToolInvocationCard supports live streaming output, elapsed time counters, syntax-highlighted terminal output, clickable URLs, copy-to-clipboard, and rich output rendering. The AgentRunTimeline component shows real-time DAG visualization of cerebro execution with status indicators, critic evaluations, judge verdicts, evidence panels with clickable citations, and run statistics (tokens, cost, duration). **RunReplayViewer** (`client/src/components/agent/RunReplayViewer.tsx`) provides step-through replay of agent runs from event log with play/pause/step controls and timeline scrubber. **PlanDiffViewer** (`client/src/components/agent/PlanDiffViewer.tsx`) shows side-by-side diffs when agent replans mid-execution. Route: `/agent/replay/:runId`. **BudgetDashboard** (`client/src/components/admin/BudgetDashboard.tsx`) provides full admin cost tracking with real-time cost ticker, per-model breakdown charts, daily/weekly/monthly trends, budget utilization gauges, top-cost runs, cost alerts, and CSV export. Route: `/admin/budget`. Backend API: `GET /api/admin/budget`.

**Agent Soul & Personality** (`server/agent/soul/`): `agentIdentity.ts` (user-assignable name default "Ilia", personality traits: curious/empathetic/precise/proactive, emotional state tracking with mood inference from messages, Redis-backed persistence), `humanBond.ts` (deep user understanding: communication style detection, expertise tracking, interest extraction, schedule patterns, goal tracking, proactive suggestions, relationship depth over time), `initiative.ts` (autonomous initiative engine: opportunity detection for code quality/docs/security/performance, proactive action scheduling, daily/weekly insights, impact/effort prioritization), `index.ts` (unified SoulEngine integrating identity+bond+initiative, injects personality context into system prompts). Personality is injected into both the chat stream system prompt (`chatAiRouter.ts` via `AgentIdentity.getSystemPromptFragment()`) and the agent executor system prompt (`agentExecutor.ts` with Ilia identity prefix).

**Deep Research Agent** (`server/agent/research/`): `deepResearchEngine.ts` (multi-phase research pipeline: query decomposition into research questions, systematic literature search, evidence extraction, cross-reference verification with corroboration/contradiction/extension detection, confidence scoring with uncertainty quantification), `evidenceSynthesizer.ts` (citation chain building, contradiction detection with resolution hints, consensus mapping, knowledge gap identification, LLM-powered synthesis), `hypothesisGenerator.ts` (LLM-powered testable hypothesis generation: causal/correlational/descriptive/counterfactual types, plausibility/novelty/impact scoring, validation approaches with feasibility, causal chain extraction, counterfactual analysis).

**Continuous Self-Improvement** (`server/agent/selfImprovement/`): `performanceTracker.ts` (success/failure rates per tool/task type/complexity, automatic weak area identification, trend analysis 24h/7d/30d), `skillEvolution.ts` (5-level proficiency: novice→competent→proficient→expert→master, XP-based progression with complexity-weighted gains, skill combination unlocks), `learningLoop.ts` (post-run lesson extraction, structured knowledge patterns, RLAIF feedback signals, strategy adaptation from past outcomes).

**MCP Auto-Discovery & Skill Acquisition** (`server/agent/mcp/`): `mcpRegistry.ts` (central registry of MCP tools with usage stats, reliability scores, auto-disable for unreliable tools, search by name/tags), `mcpAutoDiscovery.ts` (continuous discovery loop: env var scanning MCP_SERVERS/MCP_SERVER_*_COMMAND/URL, real stdio/HTTP connector with JSON-RPC 2.0, tool manifest fetching, health checks), `skillAcquisition.ts` (search for relevant MCP tools, safety evaluation, human approval with auto-approve threshold, acquisition pipeline: search→evaluate→approve→install→register).

**Parallel Sub-Agent Orchestrator** (`server/agent/orchestration/`): `subAgentSpawner.ts` (spawn lightweight sub-agents with isolated context, wave-based execution, DAG execution, semaphore concurrency control, retry with backoff, abort/cancel, pool stats), `taskDecomposer.ts` (LLM-powered decomposition into micro-tasks, dependency DAG with cycle detection, execution waves via topological sort, supports 50 micro-tasks), `resultAggregator.ts` (LLM-powered synthesis, output deduplication via content hashing, conflict detection via Jaccard similarity, quality scoring, task-to-result matching by taskId/agentId/index fallback).

### Infrastructure
Security is enforced with bcrypt password hashing, multi-tenant validation, authentication middleware, agent run limits, and production-grade security headers (CSP, HSTS). Safe process execution is ensured through centralized modules with program allowlists and argument validation. SQL security for the admin query explorer includes SELECT-only validation and audit logging. Custom error classes, global Express error handling, and Zod validation middleware for APIs are implemented.

### Scalability & Performance
Enterprise-grade scalability and performance are achieved through Redis SSE Streaming for horizontal scaling, a Memory Cache Layer (LRU with optional Redis backend), Response Caching Middleware with ETag support, and Request Deduplication. Compression Middleware handles Gzip and Brotli. Circuit Breakers wrap external services with configurable timeouts and retries. Rate Limiting uses a sliding window algorithm. Graceful Shutdown ensures connection draining and WebSocket cleanup. A FastAPI SSE Backend provides a production-grade Python SSE microservice for agent tracing using Redis Streams. Robust production-grade systems include a Large Document Processor, Dialogue Manager FSM, Stage Watchdog with AbortController, Memory Leak Prevention, Tenant-Isolated Circuit Breaker, PostgreSQL Health Checks, EventStore Batch Inserts, Connection Heartbeat Manager, Context Compressor, Semantic Cache, Graceful Degradation, Self-Healing System, OpenTelemetry Distributed Tracing, Output Sanitizer, and Backtracking Manager.

### Data Storage
PostgreSQL is used as the relational database, managed with Drizzle ORM. Client-side persistence leverages `localStorage` for chat history and preferences, and IndexedDB for background tasks and the offline queue. The CSRF system uses a shared token store with cookie-first resolution and in-memory fallback.
